## 一、修复的 Bug

### Bug 1：日期解析异常（严重，阻断接口调用）

**现象**：`LocalDateTime.parse("2026-06-15", DateTimeFormatter.ofPattern("yyyy-MM-dd"))` 会直接抛出 `DateTimeParseException`，原因是 `LocalDateTime` 需要包含时间部分（`HH:mm:ss`），但前端只传了纯日期。

**影响范围**：
- `PriceCalculationController` 的 POST / GET 两个 `calculate` 方法
- `OrderServiceImpl.createOrder` 方法
- `OrderServiceImpl.checkRenewAvailability` 方法
- `OrderServiceImpl.renewOrder` 方法

**修复方案**：使用 `LocalDate.parse(...).atStartOfDay()` 替代，先解析为纯日期，再补全当天 00:00:00 时间部分：

```java
// 修复前：
LocalDateTime startDate = LocalDateTime.parse(request.getStartDate(), DateTimeFormatter.ofPattern("yyyy-MM-dd"));

// 修复后：
LocalDateTime startDate = LocalDate.parse(request.getStartDate(), DateTimeFormatter.ofPattern("yyyy-MM-dd")).atStartOfDay();
```

**修改文件**：
- `PriceCalculationController.java` (4 处，L42-43, L85-86)
- `OrderServiceImpl.java` (4 处，L50-51, L133, L181)

---

### Bug 2：页面回显缺失（接口失败时价格区留白）

**现象**：当后端接口不可用或返回非 200 时，前端 `priceResult` 被设为 `null`，价格明细区只显示"租用天数"而没有总价，用户无法得知估算价格。

**修复方案**：
1. 新增 `getLocalFallbackPrice()` 本地兜底计算方法，接口失败时用 `vehicle.price × days` 做本地估算。
2. 兜底结果增加 `_fallback: true` 标记，UI 顶部显示橙色提示"服务端暂不可用，以下为本地估算（优惠折扣可能未计入）"。
3. `useEffect` 依赖增加 `vehicle`，确保车辆信息异步加载完成后、若日期已设置能重新触发价格试算（解决从订单页"复租"跳转后 dates 先设置、vehicle 后加载导致的价格不刷新问题）。

**修改文件**：
- `VehicleDetail.tsx`：新增 `getLocalFallbackPrice()`，`fetchPriceCalculation()` 增加 catch fallback，`useEffect` 依赖补 `vehicle`，价格 UI 增加 `_fallback` 提示条和金额 `Number()` 强制转换。

## 二、计费规则确认

| 规则 | 条件 | 折扣率 | 应用阶段 |
|------|------|--------|----------|
| 7 天长租优惠 | `rentalDays ≥ 7 且 < 30` | 95 折（减免 5%） | 在 basePrice 上直接减免 |
| 30 天长租优惠 | `rentalDays ≥ 30` | 90 折（减免 10%） | 在 basePrice 上直接减免（与 7 天互斥） |
| 企业用户折扣 | `userType = "enterprise"` | 90 折（减免 10%） | 在长租优惠后的金额上叠加计算 |

**租期计算**：
- 新订单：`ChronoUnit.DAYS.between(start, end) + 1`（首尾都算，含当天取、当天还）
- 续租：`ChronoUnit.DAYS.between(currentEnd, newEnd)`（不含 currentEnd，因为当天已在原订单计费）

**舍入模式**：`RoundingMode.HALF_UP`（四舍五入至分）

### 3.2 企业用户（userType = "enterprise"）

#### 场景 D：短租 3 天（仅企业折扣）

- **预期 totalPrice = 897 - 89.70 = 807.30**
- **预期 totalDiscount = 89.70**

#### 场景 E：7 天长租（7 天优惠 + 企业折扣叠加）

#### 场景 F：30 天长租（30 天优惠 + 企业折扣叠加）

- 企业折扣额 = 8073 × (1 - 0.90) = **807.30**
- **预期 totalPrice = 8970 - 897 - 807.30 = 7265.70**
- **预期 totalDiscount = 897.00 + 807.30 = 1704.30**

## 五、用户类型识别链路

```
前端 localStorage.token
    ↓ axios 拦截器（App.tsx L57-61）
    ↓ HTTP Header: Authorization: Bearer <token>
    ↓ PriceCalculationController#calculatePrice
    ↓ JwtAuthHelper.getCurrentUser()  → 解析 username → 查 users 表
    ↓ user.userType ∈ {"personal", "enterprise"}
    ↓ 未登录则 fallback 为 "personal"
    ↓ PriceCalculationServiceImpl#calculate
```
