# 车辆搜索与筛选逻辑修复记录

统一车辆列表页和地图视图页的搜索筛选逻辑，确保相同筛选条件下两个页面返回一致的结果集合。

## 统一的筛选参数

两个页面现在使用完全相同的筛选参数体系：

| 参数名 | 类型 | 说明 | 默认值 |
|--------|------|------|--------|
| keyword | string | 关键词（匹配名称/描述/车型） | 空 |
| city | string | 城市筛选 | 空 |
| type | string | 车型筛选 | 空 |
| minPrice | number | 最低价格 | 0 |
| maxPrice | number | 最高价格 | 2000 |
| available | boolean | 可租状态 | true |
| sortBy | string | 排序字段（price/rating/name） | rating |
| sortOrder | string | 排序方向（asc/desc） | desc |

## 验证方法

在两个页面分别设置相同的筛选条件，比对结果数量和顺序是否一致：

1. 关键词搜索：输入"电动"，两页应都只显示电动车
2. 城市筛选：选择"北京"，两页应都只显示北京的车辆
3. 价格区间：拖动滑块到 300-500，两页结果应一致
4. 可租状态：选择"已租满"，两页都应只显示不可租车辆
5. 组合筛选：城市+车型+价格，结果数量和排序应完全一致
