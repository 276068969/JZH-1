# 地图页定位失败与演示数据处理修复记录

## 修复概述

修复地图页在定位失败/未连接后端时，仍然展示固定演示车辆卡片、未明确告知用户数据性质的问题。优化用户感知，避免用户将 mock 演示数据误认为真实车源。

---

## 问题描述

| 场景 | 问题表现 | 风险 |
|------|---------|------|
| 定位失败 | 页面上方显示"定位失败"提示，但下方车辆区仍然展示 12 辆演示车辆 | 用户可能误以为这些是真实附近车辆 |
| 后端未启动 | Mock 数据静默兜底，无任何标识 | 用户无法区分真实数据与演示数据 |
| 无定位 + 无筛选 | 默认展示北京为中心的全部演示车辆 | 新用户首次进入页面看到虚假车源 |

---

## 修复方案

### 1. 新增 `isDemoData` 状态标记

在 `MapView.tsx` 中新增 `isDemoData` 状态变量，用于追踪当前车辆数据是否来自本地 mock 兜底：

- **API 请求成功**：`setIsDemoData(false)`
- **catch 分支（mock 兜底）**：`setIsDemoData(true)`

### 2. 新增 `hasActiveFilter` 派生状态

判断用户是否主动进行了筛选操作：

```typescript
const hasActiveFilter = useMemo(() => {
  return !!(selectedCity || searchText || typeFilter ||
    minPrice > 0 || maxPrice < 2000 || availableFilter !== 'true')
}, [selectedCity, searchText, typeFilter, minPrice, maxPrice, availableFilter])
```

**筛选条件**：
- 选择了城市
- 输入了搜索关键词
- 选择了车型
- 调整了价格区间
- 修改了可租状态筛选

### 3. 三级展示策略

根据定位状态 + 数据来源 + 筛选状态，展示不同的页面状态：

| 场景 | 展示内容 |
|------|---------|
| ✅ 真实数据 | 正常车辆卡片列表，无额外标识 |
| ⚠️ 演示数据 + 有筛选 | 显示「演示数据」Alert 提示 + 卡片右上角「演示」标签 |
| ❌ 演示数据 + 无筛选 + 无定位 | 显示引导空状态（"请开启定位或选择城市"），**不展示车辆卡片** |

### 4. 演示数据标识

在以下位置增加「演示数据」标识：

| 位置 | 形式 | 说明 |
|------|------|------|
| 列表标题栏 | `Tag` 标签 | 橙色 + 信息图标 |
| 每张车辆卡片 | 右上角角标 | 橙色胶囊「演示」字样 |
| 地图 Popup | `Tag` 标签 | 车辆名称下方 |
| 列表顶部 | `Alert` 警告 | 当 `isDemoData && !hasActiveFilter` 时显示 |

### 5. 引导空状态

当 `isDemoData && !hasActiveFilter && !hasRealLocation` 时，车辆列表区域展示引导卡片，而不是假数据：

- 图标：🚗
- 标题：尚未找到附近车辆
- 说明：请开启定位，或选择一个城市来查看可用车辆
- 按钮：「开启定位」（主按钮）、「选择城市」（次按钮）

### 6. 定位失败 Alert 文案优化

删除原文案中"当前列表按评分排序展示"的表述，避免暗示列表数据是真实的。

## 状态流转图

```
页面加载
  │
  ├─► 尝试定位
  │     ├─ 成功 → hasRealLocation = true
  │     │           └─► 请求 API → 成功 → 真实数据列表
  │     │                              └─ 失败 → mock 数据 + 演示标识
  │     │
  │     └─ 失败 → locationFailed = true
  │                 └─► 显示定位失败 Alert
  │                       └─► 有筛选 → 展示 mock 数据 + 演示标识
  │                       └─► 无筛选 → 展示引导空状态（不展示假车）
  │
  └─► 请求 API
        ├─ 成功 → isDemoData = false（真实数据）
        └─ 失败 → isDemoData = true（演示数据）
```

## 验证方法

1. **模拟定位失败**：在浏览器中拒绝定位权限
   - ✅ 定位标签显示「定位失败，点击重试」（红色）
   - ✅ 列表顶部显示黄色「当前为演示数据」Alert
   - ✅ 若未选择任何筛选条件，车辆区显示引导空状态，不展示车辆卡片

2. **选择城市后**：在定位失败状态下选择"北京"
   - ✅ 展示北京市的演示车辆
   - ✅ 每张卡片右上角有橙色「演示」角标
   - ✅ 标题栏有「演示数据」Tag
   - ✅ 地图 Marker 弹出的 Popup 中有「演示数据」Tag

3. **定位成功后**：允许浏览器定位
   - ✅ 定位标签显示「已定位」（绿色）
   - ✅ 「演示数据」标识消失（若后端返回真实数据）
   - ✅ 车辆按距离排序，显示距离胶囊

4. **后端未启动**：断开后端服务
   - ✅ 页面降级到 mock 数据
   - ✅ 所有演示数据位置均有明确标识
   - ✅ 用户不会误将演示数据当真
